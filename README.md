# Cloud Security Risk Assessment & GRC Simulation – Azure

## Overview

This project demonstrates a cloud security risk assessment conducted on an Azure-hosted virtual machine from a Governance, Risk, and Compliance (GRC) perspective. The objective of the assessment was to identify cloud security risks, evaluate vulnerabilities, analyze realistic threat scenarios, and recommend security controls aligned with industry best practices and compliance standards.

The assessment focused on common cloud security weaknesses such as exposed remote access, weak authentication controls, insufficient monitoring, and default system configurations that could increase organizational risk if left unaddressed.

## Environment & Scope

* Microsoft Azure Virtual Machine
* Single cloud-hosted system assessment
* Governance, Risk, and Compliance (GRC) focused review
* No additional cloud services included in scope

## Skills Demonstrated

* Cloud Security Assessment
* Governance, Risk, and Compliance (GRC)
* Risk Analysis
* Threat & Vulnerability Assessment
* Security Control Evaluation
* Cloud Security Best Practices
* Azure Security Fundamentals
* Security Recommendations & Mitigation Planning

## Assessment Activities

* Identified critical cloud assets and potential threat scenarios
* Evaluated security vulnerabilities and cloud misconfigurations
* Assessed risk levels based on likelihood and potential impact
* Reviewed security controls and compliance alignment
* Developed mitigation strategies to reduce identified risks

## Key Findings

* Remote Desktop Protocol (RDP) exposure increased attack surface risk
* Multi-factor authentication (MFA) was not fully implemented
* Default operating system hardening presented security concerns
* Limited centralized monitoring and alerting visibility was identified
* Several findings were classified as medium to high risk

## Recommendations

* Restrict RDP access using Azure Network Security Groups (NSGs)
* Enable MFA for all administrative accounts
* Apply regular operating system patches and updates
* Implement centralized monitoring using Azure Monitor or Microsoft Defender
* Use Role-Based Access Control (RBAC) for resource management

## Conclusion

This project demonstrates practical experience performing cloud security risk assessments and applying GRC concepts within a Microsoft Azure environment. It highlights the importance of proactive risk management, secure cloud configurations, and layered security controls to strengthen cloud security posture.
